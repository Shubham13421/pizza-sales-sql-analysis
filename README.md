# 🍕 Pizza Sales SQL Analysis

## 📌 Project Overview

This project analyzes a pizza sales dataset using SQL to uncover key business insights related to customer behavior, sales performance, and product trends. The goal is to demonstrate strong SQL skills including joins, aggregations, and data-driven decision making.

---

## 🎯 Objective

* Analyze sales data to identify top-performing pizzas
* Understand customer ordering patterns
* Evaluate revenue contribution by different categories and sizes
* Generate actionable business insights

---

## 🗂️ Dataset Description

The dataset consists of the following tables:

* **orders** → Contains order ID, date, and time
* **order_details** → Contains order-wise pizza quantities
* **pizzas** → Contains pizza ID, size, and price
* **pizza_types** → Contains pizza name, category, and ingredients

---

## 🛠️ Tools & Technologies

* SQL (MySQL / PostgreSQL)
* GitHub for version control

---

## 🔍 Key Analysis Performed

### 1. Order Analysis

* Total number of orders
* Orders by date and time

### 2. Revenue Analysis

* Total revenue generated
* Revenue contribution by pizza size

### 3. Product Performance

* Top-selling pizzas by quantity
* Top pizzas by revenue
* Category-wise performance

### 4. Time-based Trends

* Peak order hours
* Sales distribution across different times of the day

---

## 📊 Key Insights

* 🍕 **Classic category pizzas** are the most popular among customers
* 💰 **Large size pizzas** contribute the highest to overall revenue
* ⏰ **Peak sales occur during evening hours**, indicating high demand during that time
* 🏆 Certain pizzas consistently outperform others, highlighting strong customer preferences
